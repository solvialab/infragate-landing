How it works

From zero to running cluster in three steps

1

Choose a template or go custom

Admins pre-define approved configurations — K8s version, VM shape, node pools, tier. Engineers select one and the form locks to those values. Power users can go custom within their resource limits.

Template lock mode
2

Watch Terraform provision live

Click deploy and watch every Terraform step stream to your browser in real time. VCN, subnets, security lists, node pools, kubeconfig — all created automatically. Cost estimate shown before you commit.

Live SSE streaming Cost preview
3

Scale, monitor, or let it expire

Scale node pools on demand. Track per-cluster costs. Set TTL for automatic cleanup so dev clusters don't run forever. Protected production clusters can't be destroyed without admin override.

TTL auto-expiry FinOps
Platform capabilities

Everything your platform team needs

FinOps from day one

Live cost estimates on every surface — deploy form, cluster cards, detail page, admin panel. Pricing uses OCI Pay-As-You-Go rates with optional contract overrides.

Guardrails, not gatekeepers

Per-user resource limits, destroy protection on production clusters, TTL auto-expiry on dev clusters, RBAC-restricted templates. Engineers move fast, safely.

Full audit trail

Every deploy, scale, and destroy is logged with user, timestamp, duration, and status. Admin panel shows all clusters across all users.

Template-driven provisioning

Admin-defined templates encode K8s version, shape, pools, tier, TTL, and protection. Selecting a template locks the deploy form — no accidental misconfiguration.

Destroy protection

Protected clusters cannot be destroyed by regular users. Admins can override with force-destroy. TTL auto-expiry ensures dev clusters don't run forever.

Any OIDC provider

Works with Keycloak, Azure AD, Okta, Google Workspace, or any OIDC-compliant IdP. No vendor lock-in. Auto-provisions users on first login from JWT claims.
