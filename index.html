Platform capabilities

Complete cluster lifecycle
at your fingertips

From one-click provisioning to multi-pool scaling, TTL expiry, and cost governance — Infragate brings enterprise control to OKE.

One-click deploy

Engineers provision fully managed OKE clusters with automatic compartment, VCN, subnet, security list, and node pool creation — no manual steps.

Multi-pool scaling

Add or remove node pools, adjust OCPU, RAM, and storage per pool from the UI. Change preview shows current vs. new values before applying.

Live cost preview

Monthly and hourly estimates update live as you configure pools. Cost visible on deploy form, dashboard cards, detail page, and admin panels.

TTL & destroy protection

Set time-to-live in hours with color-coded countdown badges (green >24h, orange <24h, red <4h). Protected clusters require admin force-destroy override.

CIDR pool management

Admin-managed /24 ranges — each cluster consumes one on deploy and releases on destroy. No overlaps, automatic allocation.

Kubeconfig + SSH keys

Download kubeconfig (OCI exec plugin) and Terraform-generated SSH private key directly from the cluster detail page.

Live Terraform streaming

Watch terraform init, plan, and apply output stream in real time via SSE during deploy, scale, and destroy.

Bring your own infra

Supply existing VCN, compartment, or subnet OCIDs via the Advanced tab to skip resource creation and wire into existing networks.

Resource limits

Two-tier limit system: global defaults + per-user overrides. Covers clusters, pools, nodes, OCPU, RAM, storage, and tier. Visual feedback when limits are reached.

Cluster templates & RBAC

Golden paths with role-based gating

Admins define pre-approved templates (K8s version, shape, image, pools, TTL, destroy protection). Assign required Keycloak or Azure AD roles so users only see templates they're authorised for. Selecting a template pre-fills and locks resource fields — users can still set cluster name, CIDR, and advanced overrides. Template values can exceed user limits since they represent admin-approved configurations.

Cost visibility & governance

Real-time cost previews, cluster-level breakdowns, and global spend dashboards — no surprises. Both server-side and client-side cost engines produce identical results.

Deploy cost summary

Hourly & monthly cost updates live as you configure node pools, shape, and tier. Also shown in the deploy plan confirm modal.

Admin spend overview

Total monthly spend across all active clusters in the admin stats bar. Shape-specific rate overrides supported for custom OCI enterprise contracts.

Admin & governance

Five dedicated admin panels

Global oversight, resource limits, configuration, templates, and audit logs — all changes take effect immediately.

All Clusters

Every cluster across all users with status, owner, CIDR, K8s version, tier, resources, cost, and age. Stats bar with total spend.

Users & Limits

Per-user overrides for any combination of limits and tier. Reset to global defaults with one click. Override badges visible.

Configuration

CIDR pool, VM shapes, K8s versions, node images (with enable/disable), global resource limits. No restart needed.

Cluster Templates

Template table with live cost preview in add/edit modal. Enable/disable toggle and permanent delete.

Audit Log

Append-only record of every deploy, scale, destroy. Filterable by user, operation, and status.

Deployment flexibility

OCI Marketplace, Helm, or any Kubernetes

  • OCI Resource Manager Stack — one-click "Launch Stack" with guided form. Creates new OKE cluster or deploys into existing infrastructure.
  • Helm chart — single chart with configurable values, ingress-nginx with OCI load balancer annotations. Supports imagePullSecrets for private registries.
  • CI/CD pipelines — GitHub Actions (GHCR) & GitLab CI (GitLab CR) included. Images tagged dev-latest and latest.
  • Single VM (k3s) on OCI Always Free — ARM-compatible, control-plane tolerations for single-node scheduling.
  • Any OIDC provider — bundled Keycloak, or connect Azure AD, Okta, Google Workspace. PKCE flow, no client secrets in frontend.
